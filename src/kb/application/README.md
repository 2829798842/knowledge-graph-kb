# Application Layer Guide

`src/kb/application/` 是知识库的应用编排层，负责把数据库、向量索引、模型提供商和导入能力组织成明确的业务动作。

## 目录职责

### `services/`

- 面向完整业务动作的应用服务
- 包括问答会话、图谱浏览、来源浏览、模型配置等
- 通常由 API 直接依赖，负责串联多个底层组件

### `search/`

- 面向查询入口的轻量服务
- 把记录、实体、关系、来源检索包装成稳定的返回结构
- 主要负责“给前端一个明确的搜索结果”

### `retrieval/`

- 面向问答检索链路的内部组件
- 包括结构化检索、向量检索、融合排序、PPR 图谱重排和共享类型
- 更偏检索流程与排序策略，不直接承担 API 入口职责

### `imports/`

- 面向导入任务的编排
- 包括导入流水线、异步执行器、任务提交与重试逻辑
- 负责把 `importing/` 产生的导入项真正写入数据库、图谱和向量索引

## 简单理解

- `services`：做一件完整业务事
- `search`：给检索接口返回结果
- `retrieval`：问答内部怎么检索和重排
- `imports`：导入任务怎么落库和执行
